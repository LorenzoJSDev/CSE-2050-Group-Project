Milestone 3 Complexity Report

Time Complexitys:

HashMap put(key, value)
    * Average Case: O(1)
    * Wosrt Case: O(n)

HashMap get(key)
    * Avergage Case: O(1)
    * Worst Case: O(n)

Merge Sort
    * Average Case: O(nlogn)
    * Worst Case: O(nlogn)

Quick Sort
    * Average Case: O(nlogn)
    * Worst Case:   O(n^2)

Merge Sort vs. Quick Sort:
Quick sort is most of the time faster and uses less memory  then merge sort but slows down a lot in worst case situations.
Merge sort is more reliable then quick sort because in all cases it has an O(n log n) time complexity however it uses more memory when sorting.
I would use quick sort if my main goal for sorting was speed and the list I was sorting was relativly small.
I would use merge sort if I cared about consitency more then speed.
